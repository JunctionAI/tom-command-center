Module 01

How Agent Memory Works

Every agent has three memory systems working together. Understanding these is the single most important thing for making the system excellent.

The Brain Stack

Every time an agent responds — scheduled task or your message — it loads its "brain" in this exact order:

  1. AGENT.md— Who am I? What are my rules?
  2. knowledge.md— What have I learned about Tom over time?
  3. session-log-yesterday— Quick summary of yesterday's interactions
  4. training/*.md— Deep domain knowledge, mental models
  5. playbooks/*.md— PROVEN patterns with real DBH data
  6. skills/*.md— General best practices
  7. intelligence/*.md— Latest research (newest first)
  8. CONTEXT.md— What's happening RIGHT NOW
  9. shared/strategy/— 90-day plan (PREP, Oracle, Meridian, Beacon)
  10. pending events— Cross-agent events waiting to be consumed

Priority Rule

Playbooks beat skills. Skills beat general knowledge. If a playbook says "X works" based on real DBH data, the agent follows the playbook.

The Three Memory Systems

1. CONTEXT.md — Short-Term Memory

FOUNDATION (top section) — Manually written by you or Claude Code. Revenue baselines, current goals, active campaigns. You maintain this.

LIVE UPDATES (bottom section) — Auto-populated from [STATE UPDATE:] markers. Rolling window of last 20 updates. Old ones drop off automatically.

If CONTEXT.md gets stale, the agent operates on old assumptions. When reality shifts, update the foundation.

2. intelligence.db — Long-Term Memory

Structured learning database. Stores insights with confidence levels: EMERGING → PROVEN (3+ validations) → DISPROVEN (2+ failures).

Also stores: ROAS data, citation checks, auto-optimizer actions, cross-agent events, metrics over time.

This accumulates automatically. You don't touch it. Over time, it makes every agent smarter.

3. event_bus.db — Cross-Agent Communication

When one agent discovers something another should know, it publishes an event. PREP and Oracle see everything. Meridian sees campaign/customer/inventory events.

Fully automatic. No management needed.

Module 02

How to Give Feedback That Sticks

Your corrections are the highest-quality signal in the system. Here's how to make them count.

Rule 1: Reply in the Right Channel

Every agent has its own Telegram channel. Reply where you saw the message. The agent receives your reply with its full brain loaded and will process your feedback immediately.

Rule 2: Be Specific About What's Wrong

That was wrong
The revenue number you quoted was $34K but Shopify shows $41K. Always use Shopify as source of truth for revenue.
Don't do that again
Never suggest discounting Marine Collagen below $49. Our margin doesn't support it.

Specific feedback becomes a [STATE UPDATE:] that persists. Vague feedback gets lost.

Rule 3: Tell the Agent to Remember

If something is important enough to persist, say so explicitly:

"Remember this: Tom's workout time is ALWAYS after 7:30pm. Never suggest morning workouts."
"Save this permanently: Sea Cucumber is our #1 product by revenue, not GLM."
"Update your context: We launched the Pure Pets campaign on March 15 at $20/day."

Rule 4: Correct the Foundation When Reality Shifts

Option A: Tell the agent directly in Telegram — "Update your foundation: our monthly revenue baseline is now $50K not $40K."

Option B: Edit CONTEXT.md via Claude Code. Change the FOUNDATION section. Next time the agent wakes, it reads the new version.

When to do this: Revenue shifts, new campaigns launch, team changes, strategy pivots, personal changes.

Module 05

The Intelligence Pipeline

Understanding how data flows through the system helps you spot where things break and how to feed the machine better data.

Data Flow

External APIs — Shopify, Klaviyo, Meta, Google Ads pull raw performance data
data_fetcher.py — Formats data into text blocks for injection
orchestrator.py — Loads agent brain + injects data into prompt
Claude API — Agent generates response with full context
process_response_learning() — Extracts markers from response
Telegram — Cleaned response delivered to you

What Gets Extracted from Every Response

[STATE UPDATE:] → CONTEXT.md (rolling 20) + intelligence.db (permanent)

[EVENT:] → event_bus.db → consumed by subscribed agents next wake

[INSIGHT:] → learning.db + intelligence.db (confidence progression)

[TASK:] → Asana task auto-created with assignee and due date

[BRIEF:] → Design brief generated → briefs.db + Asana task for Roie

How Agents Learn Over Time

Agent processes data and generates insight
Insight stored as EMERGING (first observation)
Next time agent sees similar data, references stored insight
After 3+ validations → PROVEN (high confidence)
If fails 2x → DISPROVEN (never repeat)
PROVEN insights feed into monthly playbook updates
Module 06

When Things Go Wrong

Every system has failure modes. Here's how to diagnose and fix the most common ones.

Agent gives stale information
The agent's CONTEXT.md is outdated. Fix: message the agent directly with the correct info. Or edit CONTEXT.md in Claude Code. Reality shifted but the file didn't.
Agent contradicts itself
Check the playbook. If the playbook says discounting is disproven, the agent should follow it. If fresh data contradicts the playbook, it's time for a playbook update — the new data might reveal a context-dependent truth.
Auto-optimizer made a bad decision
  1. Go to command-center Telegram channel
  2. Type: undo <action_id> (the notification included the ID)
  3. Message Meridian explaining why: "Don't scale [campaign] — creative is fatiguing."
  4. The system stores your correction as a state update
ROAS tracker flags a false positive
  1. Resume the campaign manually in Meta Ads Manager
  2. Message Meridian: "GLM campaign ROAS dipped due to weekend. Resume it."
  3. If weekends consistently cause dips, we can exclude them from the 3-day rule
Agent doesn't respond
  1. Check Nexus: status — is the system running?
  2. Check Nexus: integrations — are APIs connected?
  3. Check Railway logs for errors
  4. If brain loading failed, you'll see a specific error. Check the agent's AGENT.md exists.
Tony report has wrong numbers
  1. DON'T send it to Tony yet
  2. Check Shopify admin for the real number
  3. Edit the report file: ~/dbh-aios/reports/tony-reports/tony-report-latest.md
  4. Message PREP: "Tony report had revenue wrong. Shopify shows $X not $Y."
  5. Send the corrected version

Module 08

Maintaining Quality Over Time

The system compounds — but only if you prevent the three types of decay.

1. CONTEXT.md Foundations Decay

Reality shifts faster than files get updated. A revenue baseline from 2 months ago makes every recommendation wrong.

2. Playbooks Decay

Patterns that were PROVEN last quarter may not work this quarter. Platforms change. Audiences shift. Competitors adapt.

3. Strategy Decay

The 90-day plan is a living document. When a campaign launches, a milestone hits, or a pivot happens, update it.

The Fix is Simple

When you know something the system doesn't, tell it. That's the entire quality maintenance strategy.
Module 09

How to Add New Knowledge

The system is designed to grow. Here's how to add new capabilities without breaking anything.

Add a Skill to an Agent

Write a markdown file. Save to agents/{agent}/skills/new-skill.md. Next time the agent wakes, it reads it automatically. No restart needed.

Add a Playbook

Write it with PROVEN / EMERGING / DISPROVEN sections. Save to agents/{agent}/playbooks/. Every claim must reference specific campaign data with real numbers.

Update the Strategy

Edit agents/shared/strategy/90-day-brief.md. All four strategic agents (PREP, Oracle, Meridian, Beacon) see it immediately. One file, four agents.

Add a New Agent

1. Create folder: agents/new-agent/
2. Create AGENT.md with identity + instructions
3. Add skills/ and playbooks/ folders
4. Create state/CONTEXT.md
5. Create Telegram group, add bot, note chat ID
6. Add to config/telegram.json + config/schedules.json
7. Restart on Railway

Module 10

The Philosophy

Why this system works — and why it only works if you operate it well.

Traditional Business

Hire people → manage people → people manage processes → processes produce results

Your Model

Build intelligence → intelligence manages processes → processes produce results → results improve intelligence

The bottleneck isn't execution. It's the quality of the intelligence driving execution. That's why:

Playbooks matter more than skills — proven beats theoretical, every time

CONTEXT.md matters more than AGENT.md — current state beats static identity

Your corrections matter more than defaults — you see reality, the system sees data

The Compounding Effect

Every time you correct an agent, it learns.

Every campaign that runs teaches the playbooks something.

Every month, the system knows more than it did last month.

Every night, Beacon publishes another article building your organic moat.

Every order enriches the customer intelligence database.

The system doesn't just run. It gets better. But only if you feed it truth.

Your One Rule

When you know something the system doesn't, tell it.
