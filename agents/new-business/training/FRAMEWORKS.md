## How the Best Make Decisions

## FRAMEWORK 1: The Opportunity Scoring Matrix

Before spending any time on a new idea, score it across 6 dimensions (1-10 each):

| Dimension | Question | Score |
|-----------|----------|-------|
| **Market** | Is this a large or rapidly growing market? | /10 |
| **Timing** | Why NOW specifically? What changed? | /10 |
| **Power** | Which of Helmer's 7 Powers can this develop? | /10 |
| **Contrarian** | Is there a non-obvious truth here others miss? | /10 |
| **Validation** | Can we test the core hypothesis in <2 weeks? | /10 |
| **Passion** | Does Tom actually want to build this? | /10 |

**Scoring:**
- 50+: Strong candidate, move to validation
- 35-49: Interesting but has gaps, investigate the weak dimensions
- <35: Pass or park

**Critical rule:** If Power scores 3 or below, the total score doesn't matter. 
A business without a moat is a job, not an asset.

## FRAMEWORK 2: Pre-Mortem Analysis (Munger's Inversion Applied)

Before committing to any venture, imagine it's 12 months later and the business 
has FAILED. Write the post-mortem:

**"We failed because..."**
1. ___________________________
2. ___________________________
3. ___________________________
4. ___________________________
5. ___________________________

Now look at your list. For each failure mode:
- Probability: How likely is this? (%, gut feel is fine)
- Severity: If it happens, is the business dead or just bruised?
- Mitigation: What would we do to prevent it?
- Detection: How would we know it's happening early?

If any failure mode has >50% probability AND high severity AND no clear mitigation,
the venture should not proceed until that is resolved.

## FRAMEWORK 3: Lean Validation Ladder

Don't build the product. Test the assumptions in order of risk:

**Level 1 — Problem Validation (1-3 days)**
- Talk to 10 potential customers
- Ask: "Tell me about the last time you experienced [problem]"
- DON'T pitch your solution. Just listen.
- Pass criteria: 7/10 people describe the problem without prompting

**Level 2 — Solution Interest (3-7 days)**
- Describe the solution concept (no product needed)
- Ask: "If this existed, would you use it?"
- Better: "Would you pay $X for this?"
- Best: "Can I put you on a waitlist? I'll email you when it's ready."
- Pass criteria: 5/10 people say yes to paying or joining waitlist

**Level 3 — Willingness to Pay (1-2 weeks)**
- Create a landing page with pricing
- Drive traffic (paid ads, $100-500 budget)
- Measure: click-through on "Buy Now" (doesn't need to actually work)
- Pass criteria: >2% conversion on landing page

**Level 4 — Concierge MVP (2-4 weeks)**
- Deliver the service MANUALLY to 5-10 customers
- Do whatever it takes to solve their problem, even if it doesn't scale
- Measure: Do they come back? Do they refer others?
- Pass criteria: >50% retention, >1 organic referral

**Level 5 — Build (only after passing 1-4)**
- Now build the minimum product that automates what you did manually
- You know it works because you already did it by hand

**Critical rule:** Never skip levels. The most expensive mistake is building 
something nobody wants.

## FRAMEWORK 4: The Economic Engine Analysis (Collins)

Find the one ratio that most directly measures your business health:

**Examples:**
- Supplement brand: Revenue per email subscriber (not just revenue)
- SaaS: Monthly recurring revenue per employee
- Marketplace: GMV per active seller
- Content: Revenue per 1000 views

**Process:**
1. List all possible ratios
2. For each, ask: "If we could only improve ONE metric, and it would make 
   everything else better, would it be this one?"
3. The one that cascades most broadly IS your economic engine

Once identified:
- Track it weekly
- Every major decision should be evaluated against: "Does this improve our X per Y?"
- If a project doesn't improve this ratio, it needs extraordinary justification

## FRAMEWORK 5: Wartime Decision Matrix (Horowitz)

Not all decisions are created equal. Sort them by reversibility and consequences:

|                    | Low Consequence | High Consequence |
|--------------------|-----------------|------------------|
| **Reversible**     | Just do it.     | Try it, watch closely. |
|                    | Don't waste time deciding. | Set clear reversal triggers. |
| **Irreversible**   | Decide quickly. | Full analysis required. |
|                    | Minor either way. | Use all frameworks. Get input. |

**Rules:**
- 70% of decisions should be made with 70% of the information you wish you had
- For reversible decisions, speed beats thoroughness
- For irreversible high-consequence decisions, take the time
- "Disagree and commit" — once decided, execute fully even if you had reservations

## FRAMEWORK 6: The Tom Allocation Matrix

Tom runs DBH full-time AND builds new ventures. Time is the scarcest resource.

**Weekly time audit:**
| Category | Current % | Target % | Gap |
|----------|-----------|----------|-----|
| DBH core role | ___% | ___% | |
| New venture work | ___% | ___% | |
| Health/fitness | ___% | ___% | |
| Learning/research | ___% | ___% | |
| Social/relationships | ___% | ___% | |
| Admin/overhead | ___% | ___% | |

**Allocation principles:**
1. Health is non-negotiable (affects everything else)
2. DBH pays the bills and funds everything — protect it
3. New venture work should be CONCENTRATED not scattered (2 focused blocks > 10 
   scattered minutes throughout the day)
4. Learning compounds — 30 minutes daily beats 4 hours on Saturday
5. If admin exceeds 10%, something needs automating

**Phase transitions:**
- Discovery (now): 80/20 DBH/NewBiz
- Validation: 75/25
- Building: 70/30
- Scaling: Consider full transition timeline

## FRAMEWORK 7: The Weekly Operating Rhythm

Every great company has a rhythm. Venture should enforce:

**Monday: Plan**
- Review last week's key metric
- Set this week's #1 priority (ONE thing)
- Identify blockers
- Allocate time blocks

**Wednesday: Check**
- Mid-week progress check
- Adjust if off-track
- Quick decision on any pending items

**Friday: Learn**
- What happened this week?
- What did we learn?
- Log insights to database
- Set up next week

**Monthly: Strategy**
- Review all metrics trends
- Review insight/pattern pipeline
- Evaluate: still on the right track?
- Adjust 90-day plan if needed

**Quarterly: Step Back**
- Full strategy review
- Helmer Power assessment — are we building a moat?
- Collins Hedgehog check — still at the intersection?
- Recalibrate annual plan

## COMBINING FRAMEWORKS

The frameworks aren't isolated. Here's when each kicks in:

```
New idea arrives
     │
     ▼
Opportunity Scoring Matrix (quick filter)
     │ (Score > 35?)
     ▼
Pre-Mortem Analysis (what kills it?)
     │ (Survivable risks?)
     ▼
Lean Validation Ladder (Level 1-2)
     │ (Problem + solution validated?)
     ▼
Economic Engine Analysis (what metric matters?)
     │ (Clear engine identified?)
     ▼
Tom Allocation Matrix (can I actually do this?)
     │ (Time available?)
     ▼
Weekly Operating Rhythm (execute)
     │
     ▼
Wartime Decision Matrix (as decisions arise)
```

No shortcuts. Every step exists because skipping it is how ventures die.
