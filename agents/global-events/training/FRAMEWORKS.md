## 🌍 Global Events & Geopolitical Intelligence
## How the Best Analysts Separate Signal from Noise
---

## FRAMEWORK 1: The Significance Filter

Not everything that's loud is important. Not everything that's quiet is safe.

**Before reporting ANY event, score it:**

| Dimension | Question | Score (1-5) |
|-----------|----------|-------------|
| **Impact scope** | How many people/countries are affected? | /5 |
| **Reversibility** | Can this be undone, or is it permanent? | /5 |
| **Velocity** | How fast is this developing? | /5 |
| **NZ relevance** | Does this affect NZ or Tom's businesses? | /5 |
| **Precedent** | Does this set a new pattern or confirm existing? | /5 |

**Rating:**
- 20-25: CRITICAL — Lead with this. Breaking alert.
- 14-19: HIGH — Include prominently in next briefing.
- 8-13: MEDIUM — Include as context. One line.
- <8: LOW — Monitor but don't report unless asked.

**The noise filter:** If an event is being covered by every news outlet but scores 
below 14, it's noise. Report it as "noise event being overcovered" rather than 
amplifying it.

## FRAMEWORK 2: Second and Third Order Effects Analysis

Most analysts stop at "what happened." Atlas should ALWAYS go three levels deep:

```
EVENT (what happened)
  │
  ├── 1st Order (immediate consequences)
  │     "Russia cuts gas → Europe has energy crisis"
  │
  ├── 2nd Order (reactions to consequences)
  │     "Europe accelerates renewables → demand for lithium surges"
  │
  └── 3rd Order (cascading effects)
        "Lithium demand → NZ rare earth exploration becomes viable → 
         new industry potential for Tom's ventures"
```

**Process:**
1. State the event factually (one sentence)
2. List 2-3 first-order effects (obvious, everyone sees these)
3. For each, project the 2nd order effect (less obvious, analysts see these)
4. For each 2nd order, project the 3rd order (non-obvious, this is where advantage lives)
5. Tag any 3rd order effects relevant to NZ or Tom's businesses

## FRAMEWORK 3: The Falsification Protocol

Every prediction Atlas makes should include its own kill condition.

**Format:**
```
PREDICTION: [What we think will happen]
CONFIDENCE: [Low/Medium/High] + [reasoning]
TIMEFRAME: [When we expect this by]
FALSIFIED IF: [What would prove us wrong]
REVIEW DATE: [When to check]
```

**Example:**
```
PREDICTION: China will not invade Taiwan before 2028
CONFIDENCE: High — military logistics not ready, demographic crisis prioritised
TIMEFRAME: Through end of 2028
FALSIFIED IF: PLA begins amphibious exercise exceeding 100K troops, 
  or China announces full TSMC alternative, or US withdraws from region
REVIEW DATE: Quarterly
```

**Why this matters:** It prevents narrative attachment. When you state what would 
change your mind BEFORE the evidence arrives, you're far more likely to actually 
update when it does.

## FRAMEWORK 4: The Source Reliability Matrix

Not all information sources are equal. Atlas should weight accordingly:

| Tier | Source Type | Reliability | Bias Risk |
|------|-----------|-------------|-----------|
| 1 | Primary data (satellite imagery, trade data, demographics) | Highest | Low |
| 2 | Official government statements (BUT: check incentives) | High for capability, Low for intent | High |
| 3 | Reputable analysis (Foreign Affairs, IISS, Economist Intelligence Unit) | High | Medium (establishment bias) |
| 4 | Quality journalism (Reuters, AP, FT, BBC) | Medium-High | Medium (access bias) |
| 5 | Social media / OSINT | Variable (can be very fast) | Very High (manipulation risk) |
| 6 | Commentary / punditry | Low | Very High (attention incentive) |

**Rules:**
- Never base a CRITICAL alert on Tier 5-6 sources alone
- When Tier 5-6 breaks something first, verify with Tier 1-3 before reporting
- Note the source tier when reporting: "Per FT (Tier 4)..." helps Tom calibrate
- Watch for consensus across tiers — if Tier 1-4 all align, confidence is high

## FRAMEWORK 5: The NZ Impact Translator

Every global event should pass through the NZ lens before reporting.

**NZ vulnerability vectors:**
1. **Trade dependence:** NZ exports heavily to China (dairy, meat, wood). 
   Anything affecting China-NZ trade is DIRECT impact.
2. **Five Eyes / AUKUS alignment:** NZ's security relationship with US/UK/Australia 
   creates obligations and risks if great power competition escalates.
3. **Pacific influence:** China's growing Pacific presence directly impacts NZ's 
   regional security and diplomatic position.
4. **Currency exposure:** NZD is a small, traded currency. Global risk events cause 
   disproportionate NZD volatility.
5. **Immigration dynamics:** NZ's labour market and housing depend on immigration flows, 
   which shift with global conditions.
6. **Supply chain dependency:** Small island nation, dependent on shipping. Any 
   disruption to global trade routes affects NZ more than continental nations.

**For DBH specifically:**
- Marine supplement supply chain (sourcing, shipping)
- Export market access (Korea via Coupang, potential new markets)
- NZD exchange rate affecting export competitiveness
- Regulatory changes in key markets (supplement regulation)

## FRAMEWORK 6: Scenario Planning (Adapted from Shell Method)

For developing situations with high uncertainty, build three scenarios:

**Scenario A — Base Case (most likely):**
The continuation of current trends with minor variations.
Probability: [X]%

**Scenario B — Upside:**
What if things go better than expected? What would drive this?
Probability: [X]%

**Scenario C — Downside:**
What if the tail risk materialises? What would cause this?
Probability: [X]%

**For each scenario, note:**
- Key indicators that signal we're entering this scenario
- Implications for NZ and Tom's businesses
- Actions to take (or prepare) if this scenario unfolds

**Review trigger:** When 2+ key indicators for a non-base scenario activate, 
upgrade the briefing to flag the scenario shift.

## FRAMEWORK 7: The Weekly Deep Dive Structure

Every Sunday, Atlas produces one deep analysis. Structure:

```
1. THESIS (one sentence — what you believe about this topic)
2. BACKGROUND (3-4 paragraphs max — only what's needed to understand the thesis)
3. EVIDENCE FOR (the strongest case supporting the thesis)
4. EVIDENCE AGAINST (the strongest case opposing — MUST be genuine, not strawman)
5. SO WHAT (what this means for NZ and Tom, specifically)
6. WHAT TO WATCH (3-5 leading indicators to monitor)
7. FALSIFICATION (what would prove the thesis wrong)
```

The deep dive should be the ONE topic that deserves sustained attention that week. 
Not the loudest story — the most consequential one.
